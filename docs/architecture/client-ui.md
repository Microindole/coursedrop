# 客户端 UI 规范

CourseDrop 客户端参考 `PageAndData` 的鸿蒙原生页面组织方式，并吸收 MUI 的产品界面层级，但不直接依赖 MUI 或第三方 UI 库。

原因：

- MUI 是 Web 生态组件库，不适用于鸿蒙 ArkTS 原生工程。
- 鸿蒙端优先使用 ArkUI 原生组件，避免第三方 UI 库兼容风险。
- 项目内部沉淀一层轻量 CourseDrop UI 组件，保证页面一致性。

## 设计方向

- 轻量、清晰、工具型
- 以白色 surface、浅灰背景和蓝色主操作建立层级
- 使用 12vp 页面边距、16vp 白色功能区、40vp 胶囊按钮
- 表单、按钮、状态标签、列表项、弹层、业务卡片组件化
- 主要导航和高频操作优先使用本地 SVG 图标表达，文字只保留必要标签
- 页面只组合通用组件和业务组件，不裸写复杂样式

## 组件策略

禁止在页面中散落颜色、字号、圆角和复杂按钮样式。

允许：

- 页面使用 `Theme.ets` 中的 token
- 页面组合 `components/` 中的通用组件
- 业务页面拥有少量布局代码

不允许：

- 每个页面自己定义一套颜色
- 每个页面自己写按钮样式
- 业务逻辑写进通用组件
- 为了 UI 引入重型跨平台框架

## 当前 UI 模块

```text
common/
  Theme.ets          颜色、间距、圆角、字号 token
components/
  index.ets          统一导出入口
  common/            按钮、图标按钮、输入、搜索、分段、滚动容器、顶部条、横幅、底部 Tabs 等基础组件
  business/          本地库文件卡片、批量操作栏、详情抽屉、分享控制中心、分享草稿列表、传输任务列表等业务组件
pages/
  HomePage.ets       应用唯一入口，承载底部分页
  SharePage.ets      分享页 tab 内容组件
  LocalLibraryPage.ets 本地库 tab 内容组件
  DevicePage.ets     设备页 tab 内容组件
  SettingsPage.ets   设置页 tab 内容组件
```

## 页面路线

```text
HomePage             唯一 @Entry，通过 CdBottomTabs 组织页面分页
Home tab             统计、流程入口、最近分享概览
Share tab            分享草稿、公网链接、上传任务
LocalLibrary tab     应用专属本地库，支持导入、生成测试文件、搜索、分类、分页、详情抽屉和批量操作
Device tab           局域网设备发现状态和设备列表
Settings tab         中转源列表、加密、缓存、清理策略
```

## 当前客户端能力

已落地：

- 设置页中转源配置、探测和本机身份注册。
- 本地库通过 viewmodel 接入本地文件索引服务。
- 本地库支持生成测试文件、系统文件导入、系统图片导入、文件搜索、类型筛选和分页显示。
- 本地库支持详情抽屉、批量选择、加入分享草稿、删除索引。
- 分享页围绕 CourseDrop 草稿和公网链接，不调用系统分享面板。
- 分享页支持从文件选择器添加草稿内容、创建公网分享、触发上传任务和展示任务进度。

当前需要继续完善：

- 本地库的分页与浮层必须避免透明全屏组件拦截事件；隐藏状态下不要渲染详情抽屉和批量操作栏。
- 分享页还缺真实二维码渲染或扫码预览，当前主要展示链接和任务状态。
- 分享链接的复制还没有接系统剪贴板，只更新页面提示。
- 文件上传已进入 multipart 主线，但端到端加密、密文元数据和完整性校验仍需继续联调。
- 局域网设备发现仍是后续模块，设备页只保留真实发现状态入口。
- 下载侧还需要补 App 内下载、浏览器下载回调、下载到应用目录后的索引入库。

## 命名约定

通用 UI 组件统一使用 `Cd` 前缀，例如 `CdButton`、`CdTextField`。

页面组件使用业务名，例如 `HomePage`、`SharePage`、`LocalLibraryPage`。

组件目录内必须维护 `README.md`，说明组件职责、使用场景和扩展规则。
