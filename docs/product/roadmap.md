# 产品路线

## 当前定位

CourseDrop / 课递不是普通文件传输工具，也不是系统文件管理器。

它的定位是：

```text
本地优先的加密文件投递与分享管理工具
```

和系统自带文件传输的差异：

- 系统传输偏“附近设备之间发送文件”。
- CourseDrop 偏“围绕一次文件投递的生命周期管理”。
- 系统传输通常要求同生态、同网络或近场能力。
- CourseDrop 要支持浏览器下载、扫码登录、公网限时中转、端到端加密和过期清理。

一句话目标：

```text
让用户把文件安全、临时、可追踪地投递给另一台设备或浏览器，而不把服务器变成网盘。
```

## 第 1 阶段：公网限时投递闭环

目标：先跑通最能体现差异的能力。

范围：

- 配置中转源。
- 注册本机指纹身份。
- 本地库生成或导入真实文件。
- 文件加入分享草稿。
- 创建公网分享链接。
- 上传文件到中转服务器。
- 浏览器打开分享链接并下载。
- 过期后服务器清理。

验收：

- 鸿蒙模拟器可以创建一个公网分享。
- 电脑浏览器可以通过链接下载文件。
- 服务端不会永久保存文件。

## 第 2 阶段：扫码与身份体验

目标：把 CourseDrop 从“链接工具”推进到“扫码投递工具”。

范围：

- 分享二维码展示。
- 浏览器网页登录二维码。
- 手机 App 扫码确认登录。
- App 扫分享二维码进入下载页。
- 无账号默认使用手机指纹身份。
- 可选创建账号并绑定多个手机指纹。

验收：

- 浏览器下载受登录保护。
- 默认不需要账号即可通过手机扫码登录。
- 开启账号后可以账号密码登录。

## 第 3 阶段：端到端加密

目标：服务器只保存密文，不能理解文件内容。

范围：

- 客户端上传前加密。
- 服务端保存密文和加密元数据。
- 浏览器端 WebCrypto 解密。
- App 端下载后解密。
- 错误 key、过期、篡改都能明确失败。

验收：

- 服务器文件目录中只有密文。
- 下载方只有拿到密钥才能恢复明文。

## 第 4 阶段：App 内下载与本地库管理

目标：客户端成为完整的文件投递和接收端。

范围：

- App 扫分享链接。
- App 身份鉴权下载。
- 下载到 CourseDrop 应用目录。
- 写入本地库索引。
- 本地库按来源、分享会话、状态和过期时间管理内容。

验收：

- App 下载后的文件能在本地库看到。
- 本地库不是目录树，而是分享行为视角的管理器。

## 第 5 阶段：局域网优先直传

目标：体现“本地优先”，减少公网依赖。

范围：

- 局域网设备发现。
- 同网络设备直连传输。
- 直传失败后切换公网中转。
- 设备页展示真实设备和连接状态。

验收：

- 同局域网设备优先直传。
- 跨网络或失败场景仍可用公网限时中转。

## 第 6 阶段：诊断、稳定性和演示打磨

目标：让课程项目展示稳定、可解释、可排错。

范围：

- 诊断页展示当前中转源、身份、最近请求、最近错误、本地库根目录。
- 上传/下载失败重试。
- 分享撤回、续期、删除审计展示。
- 模拟器扫码测试说明。
- GitHub Actions 自动部署服务器。

验收：

- 出问题时能定位是网络、身份、文件、服务端还是 UI。
- 演示路径清晰：本地库 -> 分享 -> 二维码 -> 浏览器下载 -> 过期清理。
